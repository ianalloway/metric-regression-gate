# Contributing

Thanks for helping improve **metric-regression-gate**.

## Issues and pull requests

- Open an issue for larger design questions or regressions you have seen in CI.
- Keep pull requests focused: one change set per PR, with a clear description.
- Match existing style and ensure `pytest` (or the CI workflow) passes locally when you change Python code.

## Security

Please do **not** file public issues for security vulnerabilities. See [`SECURITY.md`](SECURITY.md) for contact details.
